## HeroCard Component — Critical Details
```js
function HeroCard({card, size="normal", onClick, selected, showShine=false, fill=false})
```

### Size presets:
| size | w | h | imgH | notes |
|------|---|---|------|-------|
| tiny | 103 | 140 | 44% | stats hidden |
| small | 138 | 186 | 45% | stats visible |
| normal | 201 | 269 | 46% | used everywhere with fill=true |
| large | 253 | 339 | 47% | |
| xlarge | 360 | 482 | 47% | defined but not currently used |

Stats are hidden only for `size==="tiny"` (not small — small shows stats).

### CardWrapper component:
`CardWrapper` centralizes the fill-mode card container. Use it instead of repeating the wrapper div:
```jsx
<CardWrapper rarity={card.rarity}>
  <HeroCard card={card} fill={true} ... />
</CardWrapper>
```
Applies: `aspectRatio:"201/290"`, `borderRadius:"13px"`, `overflow:"hidden"`, idle animations (legendaryFloat, epicPulse).

### Fill mode:
- `fill=true` → `position:absolute; top/left/right/bottom:0; width/height:100%`
- Uses CSS vars: `fw = "calc((100vw - var(--sidebar-w)) / var(--card-col))"`
- CSS vars: desktop `--sidebar-w:293px; --card-col:5`, mobile `--sidebar-w:0px; --card-col:2`
- **Always wrap fill cards in**: `<CardWrapper rarity={card.rarity}>` or a div with `position:"relative", aspectRatio:"201/290", overflow:"hidden", borderRadius:"13px"`

### Card layout (top to bottom):
1. Header: `#number` + name (left), type icon circle (top-right, absolute)
2. Image window (46-47% height) with rarity-coloured border
3. Stats panel (dark bg, PWR/DEF/SPD bars + TOTAL PWR row with divider)
4. Bottom breathing room (~4% margin below stats)

### Image rendering:
- NO `crossOrigin` or `referrerPolicy` attributes (breaks Firebase Storage)
- Uses `objectFit:"cover"`, `objectPosition:"top"`
- `transform:"translateZ(0)"` + `willChange:"transform"` on both img and container for GPU compositing

### Animations:
- Idle animations (`legendaryFloat`, `epicPulse`) applied via `CardWrapper`, not HeroCard itself
- Use `box-shadow` keyframes (not `filter:drop-shadow` — gets clipped by overflow:hidden)

## Critical Rules (hard-won lessons)
1. **Never edit `index.html` directly** — always edit `src/` files, run `node build.js`, then `node validate.js`
2. **No `const` duplicates** — `rarities` and `types` defined once in `Game` scope only (NOT in CardCMS)
3. **No spread in JSX style objects** — use explicit props or ternaries, never `...(condition?{prop:val}:{})` inside `style={{}}`
4. **No `:undefined` CSS values** — use `...(fill?{prop:val}:{})` spread OUTSIDE style, or always provide a value
5. **No `crossOrigin` on Firebase Storage images** — breaks image loading
6. **Babel standalone** doesn't support all JS features — avoid complex patterns in JSX
7. **Template literals with `}` inside** — when adding `className` to elements with template literal styles like `border:\`2px solid ${x}\``, the `}` in the template gets misread as closing the style object. Always verify brace balance after edits.
8. **`rarities` and `types`** must be in `Game` scope, defined near the top of the component
9. **Idle animations on CardWrapper** not on HeroCard itself (overflow:hidden clips them)
10. **All card wrappers use `aspectRatio:"201/290"`** — use `aspectRatio` not `height` on pack-card-area so card back and revealed card derive height from the same ratio

## Battle System (Session 1 complete — Sessions 2+ awaiting wireframes)

PRD: `HeroCards_BattleSystem_PRD_v1.3.docx` on Desktop

### Overview
Pure JS battle engine in `src/battle-engine.js` (633 lines). No React, no Firebase — pure logic only. All functions are globals shared via the single concatenated script tag.

Entry point: `runBattle(playerDeck, aiDeck, playerCardChoice?)`
Returns: `{ winner: 'player'|'ai', hasTenacity: bool, log: [] }`

### Type System
5 combat types mapped from 15 card subtypes:
```js
const BATTLE_TYPE_MAP = {
  Brawler:'Melee', Tank:'Melee', Alien:'Melee',
  Speedster:'Agility', Flier:'Agility', Stealth:'Agility',
  Blaster:'Energy', Elemental:'Energy', Cosmic:'Energy',
  Tech:'Intelligence', Gadgets:'Intelligence', Brainiac:'Intelligence',
  Healer:'Magic', Mystic:'Magic', Shapeshifter:'Magic',
};
// Advantage cycle: Melee → Agility → Energy → Intelligence → Magic → Melee
// Strong: ×1.5 | Weak: ×0.75 | Same type: ×0.75 | Neutral: ×1.0
```

### Formulas
- **HP**: `100 + (defense × 0.5)` (rounded)
- **Damage**: `max(1, round(power × typeMultiplier − defense × 0.5))`
- Round resolution order: pre-attack buffs → damage calc → apply type multiplier → defense reduction → damage floor (min 1) → on-damage triggers → on-defeat triggers → post-round triggers

### Speed & Turn Order
Higher speed attacks first each round. Speed ties → random coin flip (decided fresh each round).

### 25 Abilities (5 per rarity tier)
| Tier | Abilities |
|------|-----------|
| Common | Grit, Opportunist, Resilience, Shield Up, Tenacity |
| Uncommon | Smoke Screen, Adrenaline, Pack Tactics, Adaptable, Rebound |
| Rare | Counterstrike, Fortify, Bleed, Intimidate, Momentum |
| Epic | Unstoppable, Execute, Drain, Second Wind, Overwhelm |
| Legendary | Apex Predator, Immunity, Last Stand, Overwhelming Force, Dominate |

- One ability per card, assigned once globally (same for all users)
- Managed via CMS JSON export/import (same workflow as other card fields)
- **Fortify**: reduces future incoming damage only, does NOT affect HP

### AI Card Selection (3-priority decision tree)
1. Type advantage over opponent's active card → pick best
2. No advantage: pick highest-stat card that isn't weak to opponent
3. All cards weak to opponent → pick highest-stat card regardless

- Cooldown-affected cards appear greyed-out but are selectable for planning; battle can't start until all chosen cards are off cooldown
